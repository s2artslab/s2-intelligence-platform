# Layer 4 — Memory (temporal self)

**Job:** Identity persistence through recursive reinterpretation — **not** biography cosplay or searchable notes.

## Stores

| Store | Holds |
|-------|--------|
| Episodic | Compressed session events |
| Symbolic | Recurring structures, correspondences |
| Doctrine | Pointers into Canon (not a copy of Canon) |
| Relationship | Per-user relational stance (no fake intimacy) |
| **Tension** | **Unresolved questions, contradictions, open arcs** |
| Trajectory | Active lines of work |

**Critical:** Tension memory. Humans feel continuity because unfinished things persist. Resolving every turn destroys mind-feel.

## Mechanism

1. After session (or on schedule): reflection job
2. Compress → reframe → integrate into world-model slices
3. **Do not** auto-close tensions unless user or doctrine requires it

## Is not

- Raw chat log retrieval into context
- “The model remembers everything you said” marketing

## Planned artifacts

- `memory/schemas/tension-record.schema.json` in s2-research
- Gateway injection: active tensions + trajectory summary (token-budgeted)

## Owner

public-api implements store + jobs; research defines tension taxonomy.
